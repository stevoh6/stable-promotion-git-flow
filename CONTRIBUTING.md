# Contributing

Contributions are welcome when they keep the Stable Promotion Git Flow lightweight, practical, and understandable for solo developers and small teams.

## Principles

- Keep `main` / `master` clean and production-focused.
- Keep `staging/*` temporary and release-specific.
- Keep rollback simple through `release/stable`.
- Prefer practical commands over theory-only explanations.
